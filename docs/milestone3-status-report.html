2. How the PDF Export Works

USER FLOW
End-to-End Flow Login ──▶ Case Intake ──▶ Submit Case ──▶ AI Analyzes ──▶ Results Page │ ┌─────────┴─────────┐ │ │ [Download PDF] [Print] │ ▼ PDF generated on server stored securely in cloud downloaded to your device

Step-by-step:

  1. Log in at /login with your registered email and password.
  2. Fill out the Case Intake form: title, narrative, stakeholders, professional codes, consent checkboxes.
  3. Click Submit Case for Analysis. The AI analyzes the case using the 6 canonical ethical lenses.
  4. You are automatically redirected to the Results page.
  5. At the bottom of the Results page, click Download PDF.
  6. The PDF is generated on the server and automatically downloads to your device.

What the PDF contains:

  • Case title, narrative, stakeholders, and professional codes
  • Composite score, ethical stability rating, and conflict level
  • All 6 ethical lens scores with reasoning
  • Violation detection details (if applicable)
  • Professional code implications
  • Primary tensions, analytical observations, and reflection questions
  • Warning flags
  • Algorithm version used for traceability
  • Disclaimer

4. Security

The following security controls are in place for the new features:

FeatureProtection
PDF DownloadOnly the case owner (or an admin) can download a PDF. Unauthorized users receive a 403 error. No data is leaked.
PDF StoragePDFs are stored in a private bucket. Each file is accessible only to the user who generated it, via a time-limited signed URL (expires in 1 hour).
Access Code ManagementOnly admin users can view, toggle, or reset access codes. Non-admin users are blocked by row-level security policies.
Usage CountA database constraint prevents usage counts from going below zero.
PDF IntegrityEach generated PDF has a SHA-256 hash stored in the database for integrity verification.
